# 📘 SFMS Phase 1 SD - 01 공통 표준 및 CMM 상세 설계서

* **문서 버전:** v1.4 (Bento v1.1 & Icon-centric Standard)
* **최종 수정일:** 2026-03-21
* **주요 내용:** 전역 UI 표준, 공통 컴포넌트 활용, CMM 도메인 비즈니스 로직

---

## 🎨 1. 프론트엔드 UI/UX 표준 (Bento Standard v1.1)

### 1.1 레이아웃 원칙
*   **Fixed Viewport**: 모든 메인 화면은 `100vh` 내에서 작동하며 브라우저 스크롤을 발생시키지 않습니다.
*   **Independent Scroll**: 각 패널(트리, 테이블, 상세폼)은 개별적인 내부 스크롤 영역을 가집니다.

### 1.2 아이콘 및 툴팁 표준 (Icon-centric UI)
화면의 시각적 복잡도를 낮추고 데이터에 집중하기 위해 아이콘 중심의 UI를 지향합니다.
*   **원칙**: 상단 툴바의 필터, 새로고침, 추가 버튼 등은 텍스트 없이 **아이콘만 표시**합니다.
*   **가이드**: 모든 아이콘 버튼은 반드시 `Tooltip` 컴포넌트로 감싸 기능을 설명해야 합니다.
*   **아이콘셋 (Ant Design Icons)**:
    *   검색/필터: `FilterOutlined`
    *   새로고침: `ReloadOutlined`
    *   데이터 추가: `PlusOutlined`
    *   상세 보기: `CameraOutlined` (스냅샷 의미 강조)
    *   조직도 루트: `ApartmentOutlined`

### 1.3 필터 및 검색 (Floating Filter)
*   필터 영역은 기본적으로 접혀 있으며, 필터 버튼 클릭 시 슬라이드 형식으로 노출됩니다.
*   활성 상태인 필터 조건은 테이블 상단에 **풍선(Tag) 형태**로 나열하여 현재 상태를 즉시 인지하게 합니다.

## ⚙️ 2. 공통 기능 (CMM) 설계

### 2.1 공통 코드 관리
*   **Upsert 정책**: 엑셀 업로드 시 기존 코드는 업데이트하고, 없는 코드는 신규 생성하는 `Upsert` 로직을 기본으로 합니다.
*   **안전 확인**: 대량 데이터 변경 전 요약 정보(추가 건수, 수정 건수)를 사용자에게 확인받는 워크플로우를 가집니다.

### 2.2 통합 파일 관리
*   **UUID 식별**: 모든 파일은 물리적 경로 대신 `UUID`를 식별자로 관리하여 보안을 강화합니다.
*   **인증 연동**: `<img>` 태그 등 헤더 전달이 어려운 경우 URL 쿼리 파라미터 기반의 임시 토큰 인증을 지원합니다.
