# SFMS Phase 1 작업 진행 상황

## 📋 프로젝트 개요

- **목표**: 시설 유지보수 관리 시스템(SFMS) 1단계 기반 구축
- **기술 스택**: FastAPI, React 19, PostgreSQL 16, Redis, Podman
- **현재 가동 포트**: 8000 (Backend), 5173 (Frontend)

## 🛠️ 세부 진행 현황

### 1. 인프라 및 설정 (CORE)

- [x] Podman 기반 컨테이너 환경 구축 (PgSQL, Redis, MinIO)
- [x] Redis 7 인증 및 권한 설정 (Fail-open 정책 적용)
- [x] MinIO 스토리지 연동 및 버킷 설정
- [x] **DB 통합 배포 스크립트 완성** (`00_deploy.pgsql`): 계정 생성, 권한 부여, 기초 데이터 일괄 처리
- [x] **데이터베이스 백업 시스템**: 컨테이너 환경 자동 감지 및 SQL 덤프 자동화 (`backup_db.sh`)
- [x] **VSCode 설정 최적화**: Biome 중심의 포매팅/린팅 표준화 및 이식성 개선

### 2. 프론트엔드 공통 (SHARED) - ✅ 표준화 고도화 완료 (2026-03-21)

- [x] **Bento Standard v1.1 수립**: Icon-centric UI + Tooltip 패턴 도입 (텍스트 라벨 최소화)
- [x] **Smart Scroll 정책 적용**: Zero-Card-Scroll 및 10-Row Rule (테이블/트리 공통)
- [x] **글로벌 i18n 시스템 구축**: `messages.ts` 기반 Zero Hardcoded Strings + Fallback 제거
- [x] **범용 엑셀(Excel) 모듈 완성**: 멀티 시트 다운로드, 안전 업로드(확인 모달), 공통 유틸리티화
- [x] **멀티 테마 시스템**: 5가지 테마 선택 및 영속성 구현
- [x] **API 연동 최적화**: `.data.data` 접근 문제 해결 및 리프레시 토큰 자동 갱신(Interceptor) 안정화

### 3. 인증 및 권한 (IAM / USR)

- [x] JWT 기반 로그인/로그아웃 및 RTR(Rotation) 구현
- [x] 사용자 권한(Resource/Action) 기반 **동적 메뉴 필터링**
- [x] **보안 강화**: 로그아웃 시 리프레시 토큰 즉시 블랙리스트 등록 및 로그인 실패 감사 로그 기록
- [x] **시스템 관리자 정보 갱신**: ID: `admin`, PW: `admin1234`, Role: `SUPER_USER` (Password Hash 동기화)

### 4. 사용자 및 조직 관리 (USR) - ✅ Bento 표준 v1.1 적용

- [x] **조직도 관리 고도화**: `ApartmentOutlined` 기반 가상 루트("전체 조직도") 트리 구조 통일
- [x] **사용자 관리 기능 완성**: 스마트 스크롤 테이블, 드로어 기반 상세/편집, 아이콘 중심 UI 필터
- [x] **데이터 정합성**: 사번/계정 상태 관리 보강 및 더미 데이터(20여명) 구축

### 5. 공통 기능 (CMM) - ✅ Bento 표준 및 엑셀 안정화 완료

- [x] **공통 코드 관리 리팩토링**: Splitter 기반 그룹/상세 분할 레이아웃, 드로어 편집 시스템
- [x] **통합 엑셀 백업 시스템**: 전역 코드(그룹+상세) 멀티 시트 다운로드 기능 완성
- [x] **안전 업로드 워크플로우**: 덮어쓰기 경고 및 데이터 요약 확인 단계 도입

### 6. 시스템 및 감사 (SYS) - ✅ 개발 완료

- [x] **감사 로그(Audit Log) 고도화**: 상세 필터링(날짜/수행자/도메인), 페이징, 데이터 스냅샷 뷰어
- [x] **Icon-centric Filter**: 필터 토글 시스템 및 활성 필터 태그(Balloon) UI 적용
- [x] **API 테스터 도구 통합**: 표준 레이아웃 적용 및 라우팅 연동
