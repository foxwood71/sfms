# SFMS Phase 1 작업 진행 상황

## 📋 프로젝트 개요

- **목표**: 시설 유지보수 관리 시스템(SFMS) 1단계 기반 구축
- **기술 스택**: FastAPI, React 19, PostgreSQL 16, Redis, Podman
- **현재 가동 포트**: 8000 (Backend), 5173 (Frontend)

## 🛠️ 세부 진행 현황

### 1. 인프라 및 설정 (CORE) - ✅ 완료

- [x] Podman 기반 컨테이너 환경 구축
- [x] **DB 아키텍처 대개편**: Phase 기반 넘버링 및 통합 배포 마스터 스크립트 구축
- [x] **데이터베이스 통합 배포 성공**: Phase 0~4까지 오류 없이 재배포 완료 (2026-03-23)

### 2. 프론트엔드 공통 (SHARED) - ✅ 표준화 고도화 (v1.1 Refined)

- [x] **Single Bento Box**: 전 페이지 단일 카드 컨테이너 레이아웃 적용
- [x] **Solid Header Bar**: 56px 높이 고정 및 배경색 분리를 통한 헤더 볼륨감 확보
- [x] **Perfect Centering**: Empty 상태 데이터 정중앙 배치 로직 완결
- [x] **Splitter Restriction**: 좌측 패널 조절 범위(15%~40%) 제한

### 3. 인증 및 권한 (IAM / USR) - ✅ 완료

- [x] JWT RTR 및 동적 메뉴 필터링
- [x] **권한 데이터 교정**: 슈퍼유저 권한 형식(`{"ALL": ["*"]}`) 백엔드 엔진과 동기화
- [x] **인증 정보 연동**: `/auth/me` 호출 시 상세 권한 및 역할 정보 로드 오류 해결

### 4. 사용자 및 조직 관리 (USR) - ✅ 로직 완결

- [x] **말단 노드 선택 문제 해결**: 재귀 탐색(`findOrgInTree`)을 통한 하위 부서 로드 100% 보장
- [x] **ID 0 노드 보호**: 시스템 관리자 노드 수정/삭제 및 하위 생성 제한 UX 적용
- [x] **생성 로직 보강**: 부서 선택 시 해당 부서 하위로 `parent_id` 자동 주입 기능 수정

### 5. 공통 기능 (CMM) - ✅ 완료

- [x] **공통 코드 규격화**: `code_length`, `is_seq_used` 필드 추가 및 UI 연동
- [x] **상세 코드 조회 수정**: 백엔드 스키마(`CodeGroupRead`) 하위 상세 코드 필드 추가 및 프론트엔드 파싱 로직 동기화
